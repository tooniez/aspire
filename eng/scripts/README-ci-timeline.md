# CI Timeline Summary

Generates a visual timeline report for GitHub Actions workflow runs, showing job
execution breakdown, critical path analysis, and queue hotspots.

## Overview

The report shows:

- **Summary stats** — wall time, job counts, status breakdown, runner distribution
- **Critical path** — the longest-running jobs by total pipeline time (deps + queue + run)
- **Queue hotspots** — jobs that waited longest for a runner
- **Full timeline** — all jobs organized by phase

Runner labels are shown as emoji: 🐧 Linux, 🪟 Windows, 🍎 macOS (⚡ for large runners).

## Local usage

Requires the `gh` CLI (authenticated) and .NET 10 SDK.

```bash
# Generate for a specific run (opens in browser)
dotnet run --project tools/GenerateCITimeline/GenerateCITimeline.csproj -- --run-id 23783420243

# Save to an HTML file
dotnet run --project tools/GenerateCITimeline/GenerateCITimeline.csproj -- --run-id 23783420243 -o report.html

# Filter to jobs with total > 8 minutes
dotnet run --project tools/GenerateCITimeline/GenerateCITimeline.csproj -- --run-id 23783420243 --min-total 8

# Use a different repo
dotnet run --project tools/GenerateCITimeline/GenerateCITimeline.csproj -- --run-id 12345 --repo owner/repo

# Load from a pre-fetched JSON file (offline)
dotnet run --project tools/GenerateCITimeline/GenerateCITimeline.csproj -- --json ci-data.json --min-total 10
```

When no `-o` is specified and `$GITHUB_STEP_SUMMARY` is not set, the report is
written to a temp file and opened in the default browser.

## CI integration

The tool runs automatically in the `results` job of `.github/workflows/tests.yml`.
It appends the timeline report to `$GITHUB_STEP_SUMMARY` alongside the existing
test summary generated by `GenerateTestSummary`.

The CI step uses `--min-total 8` to keep the output compact by filtering out jobs
that took less than 8 minutes total.

## How it works

1. Fetches all jobs for a workflow run via `gh api` (with pagination)
2. Groups matrix variants (e.g., `Build (ubuntu-latest)` / `Build (windows-latest)`)
3. For each job, computes:
   - **Deps time**: seconds from workflow start until the job was created
   - **Queue time**: seconds from job created until it started running
   - **Run time**: seconds from started to completed
   - **Total**: seconds from workflow start to job completion (= deps + queue + run)
4. Classifies jobs into phases (Setup, Build, Tests, Templates, Validation)
5. Renders HTML tables

## Output format

- In CI (`$GITHUB_STEP_SUMMARY` set): writes pure HTML that renders in GitHub's
  step summary. No `<script>` or `<style>` tags — GitHub strips those. Uses emoji
  and inline HTML tables.
- Locally (`-o report.html`): wraps in a full HTML document with GitHub-like CSS.
- Locally (no `-o`): same as above, auto-opened in the browser.

## Parameters

| Flag | Default | Description |
|------|---------|-------------|
| `--run-id` | `$GITHUB_RUN_ID` | Workflow run ID to analyze |
| `--repo` | `$GITHUB_REPOSITORY` or `microsoft/aspire` | Repository (owner/repo) |
| `--json` | — | Load job data from a JSON file instead of the API |
| `--min-total` | `0` | Only show jobs with total pipeline time above N minutes |
| `-o` | — | Output file path (`.html` for standalone, otherwise raw) |
| `--open` | auto | Force open in browser after writing |
