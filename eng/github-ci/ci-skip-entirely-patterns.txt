# CI skip-entirely patterns -- files whose changes skip the ENTIRE CI workflow.
#
# This is the top-level skip gate, NOT the selective-test router. It lists glob
# patterns for files whose changes do NOT require any CI to run (e.g. documentation,
# non-build pipeline scripts, or specific workflow files unrelated to the build and
# test process). For path -> specific test/job routing, see test-trigger-map.yml.
#
# When EVERY file changed in a pull request matches at least one pattern here, the
# entire CI workflow is skipped (no build, no tests).
#
# Pattern syntax:
#   **   matches any path including directory separators (recursive)
#   *    matches any characters except a directory separator
#   .    is treated as a literal dot (no escaping needed)
#   All other characters are treated as literals.
#
# Lines starting with '#' and blank lines are ignored.

# This file itself - changing CI-skip patterns doesn't require a CI run.
# Note: this also means a syntax error introduced here won't be caught by CI,
# so take care when editing. Pattern conversion is validated by the
# check-changed-files action at runtime.
#
# Interaction with selective CI: eng/github-ci/test-trigger-map.yml routes a change to this
# file to Infrastructure.Tests (its TestTriggerMap suite reads this file as the
# selector's prefilter source). That route only fires when this file changes
# ALONGSIDE other non-skippable work, because the entry below makes a PR that
# touches ONLY this file match the skip gate and skip the whole tests workflow
# before the selector runs. That is intentional - editing the skip list alone
# doesn't warrant a full CI run - but it means the Infrastructure.Tests route is
# best-effort for this one file, not guaranteed.
eng/github-ci/ci-skip-entirely-patterns.txt
