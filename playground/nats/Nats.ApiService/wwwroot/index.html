NATS Aspire Playground

This api service exposes HTTP endpoints that drive a NATS server (with JetStream) running in a container. The events stream (subject events.>) is created at startup. A separate backend service subscribes directly to events.> and logs each message.

Try it: start a consumer below, then publish events and watch them arrive over Server-Sent Events. You can also open the backend resource in the Aspire dashboard's Console tab to see the same events being processed there.

Docs: NATS, NATS .NET client.
